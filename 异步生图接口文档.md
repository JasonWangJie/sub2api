# 异步生图接口文档

面向下游对接方的异步生图 API 说明。系统提供两套互不互通的异步能力，请勿混用路径、任务 ID 前缀与响应结构。

| 体系 | 任务 ID 前缀 | 存储 | 说明 |
|---|---|---|---|
| **持久化异步生图（推荐）** | `asyncimg_*` | PostgreSQL + 队列 + 对象存储 | BB / SC 方言，见下文第 1–9 节 |
| **旧版 Redis 异步** | `imgtask_*` | Redis + 对象存储 | OpenAI / Grok，见第 10 节 |

本文以**持久化异步生图**为主。更细的内部实现说明见 `docs/DURABLE_ASYNC_IMAGE_API.md`。

---

## 1. 接口一览

| 方言 | 场景 | 方法 | 路径 | 提交 HTTP |
|---|---|---|---|---|
| BB | Gemini 文生图 / 图生图 | `POST` | `/v1/chat/completions_gm` | `202` |
| BB | OpenAI 文生图 | `POST` | `/v1/images/generations_oa` | `202` |
| BB | OpenAI 图生图 | `POST` | `/v1/images/edits_oa` | `202` |
| BB | 查询任务 | `GET` | `/v1/images/tasks_async/{task_id}` | `200` |
| SC | 上传参考图 | `POST` | `/v1/uploads/images_sc` | `200` |
| SC | Gemini 文生图 / 图生图 | `POST` | `/v1/images/generations_sc` | `200` |
| SC | 查询任务 | `GET` | `/v1/tasks_sc/{task_id}` | `200` |

说明：

- 上述路径为固定协议，**无**省略 `/v1` 的别名。
- `/v1/chat/completions`、`/v1/images/generations`、`/v1/images/edits` 仍是同步接口，行为不变。
- BB / SC 是下游请求与响应方言，不是上游供应商。实际走 Gemini 还是 OpenAI，由 API Key 所属分组决定。
- SC 仅支持 Gemini 分组；OpenAI 请使用 `_oa` 路径。
- 本期仅支持图片，不含视频。

---

## 2. 启用条件

须同时满足：

1. 有效 API Key，且已分配分组。
2. 平台匹配：`_gm` / `_sc` → Gemini；`_oa` → OpenAI。
3. 分组同时开启「图片生成」与「异步生图」（`allow_image_generation`、`allow_async_image_generation`）。后者默认关闭。
4. 全站对象存储已启用且可用。
5. 余额 / 订阅 / 额度 / 并发等既有检查通过。

未开启异步生图时，提交接口返回 **`403`**，**不会**回退到同步接口。

## 3. 通用约定

### 3.1 鉴权

```http
Authorization: Bearer <API_KEY>
Content-Type: application/json
```

查询任务必须使用**提交该任务的同一个 API Key**；换 Key 一律 `404`（防止探测任务是否存在）。

### 3.2 轮询

建议每 **3 秒**查询一次。提交成功响应通常包含：

```http
Cache-Control: no-store
Location: <查询地址>
Retry-After: 3
```

处理中查询同样带 `Cache-Control: no-store` 与 `Retry-After: 3`。

是否成功以响应体中的 **`status`**（SC 为 `data.status`）为准，不要仅凭 HTTP `200` 判断成功。

### 3.3 幂等（可选）

```http
Idempotency-Key: <最多 255 字节的客户端唯一值>
```

范围：同一 API Key + 同一幂等键。

| 情况 | 结果 |
|---|---|
| 路径与请求体哈希相同 | 返回首次创建的任务，不重复生成、不计费 |
| 同键但路径或请求体不同 | `409 Conflict` |
| 未带该头 | 每次提交都新建任务（网络重试可能重复计费） |

重试时请原样重发请求体（含空白与字段顺序）。

### 3.4 查询绝对地址

由配置 `async_image.public_base_url` 拼接固定查询路径。例如：

```yaml
async_image:
  public_base_url: "https://api.example.com"
```

BB 查询地址示例：

```text
https://api.example.com/v1/images/tasks_async/asyncimg_0123456789abcdef
```

未配置时返回 `/v1/...` 相对路径。该地址是 Sub2API 对外根地址，不是图片 CDN 地址。

## 4. BB / Gemini

### 4.1 文生图

```http
POST /v1/chat/completions_gm
Authorization: Bearer <GEMINI_GROUP_API_KEY>
Content-Type: application/json
Idempotency-Key: order-20260720-001
```

```json
{
  "model": "gemini-3-pro-image-preview",
  "stream": false,
  "max_tokens": 4096,
  "messages": [
    {
      "role": "user",
      "content": "现代客厅，北欧风，自然光"
    }
  ],
  "extra_body": {
    "google": {
      "image_config": {
        "image_size": "2K",
        "aspect_ratio": "16:9"
      }
    }
  }
}
```

### 4.2 图生图

参考图可为公网 HTTPS URL，或 `data:image/...;base64,...`：

```json
{
  "model": "gemini-3-pro-image-preview",
  "stream": false,
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "image_url",
          "image_url": {
            "url": "https://cdn.example.com/reference.png"
          }
        },
        {
          "type": "text",
          "text": "保留构图，把场景改成夜景"
        }
      ]
    }
  ],
  "extra_body": {
    "google": {
      "image_config": {
        "image_size": "4K",
        "aspect_ratio": "auto"
      }
    }
  }
}
```

### 4.3 参数约束

| 字段 | 规则 |
|---|---|
| `model` | 必填 |
| `stream` | 必须为 `false` 或省略 |
| `messages` | 至少一条；仅接受 `role: "user"` |
| `content` | 非空字符串，或仅含 `text` / `image_url` 的非空数组；须含非空文本提示 |
| `image_size` | 可省略，或 `1K` / `2K` / `4K`；`0.5K` 默认拒绝（仅配置白名单模型可透传） |
| `aspect_ratio` | 可省略，或 `1:1`、`2:3`、`3:2`、`3:4`、`4:3`、`4:5`、`5:4`、`9:16`、`16:9`、`21:9` |
| `auto` / `自动` | 仅图生图有效；文生图传该值返回 `400` |

### 4.4 提交响应 `202 Accepted`

```json
{
  "id": "asyncimg_0123456789abcdef",
  "task_id": "asyncimg_0123456789abcdef",
  "object": "image.task",
  "status": "queued",
  "query_url": "https://api.example.com/v1/images/tasks_async/asyncimg_0123456789abcdef"
}
```

## 5. BB / OpenAI

复用现有 OpenAI Images 参数：`model`、`prompt`、`n`、`size`、`quality`、`background`、`output_format`、`response_format` 等；`stream` 必须为 `false` 或省略。

### 5.1 文生图

```http
POST /v1/images/generations_oa
```

```json
{
  "model": "gpt-image-1",
  "prompt": "一只在沙滩上的猫，写实风格",
  "n": 1,
  "size": "1536x1024",
  "quality": "high"
}
```

### 5.2 图生图（JSON）

```http
POST /v1/images/edits_oa
```

```json
{
  "model": "gpt-image-1",
  "prompt": "保留主体，把背景换成夜景",
  "images": [
    {
      "image_url": "https://cdn.example.com/reference.png"
    }
  ],
  "size": "1024x1024"
}
```

至少提供一个 `images[].image_url`。不支持 `images[].file_id`。

### 5.3 图生图（multipart）

```bash
curl -X POST 'https://api.example.com/v1/images/edits_oa' \
  -H 'Authorization: Bearer sk-...' \
  -H 'Idempotency-Key: edit-20260720-001' \
  -F 'model=gpt-image-1' \
  -F 'prompt=把背景换成夜景' \
  -F 'image=@reference.png'
```

### 5.4 提交响应 `202 Accepted`

```json
{
  "task_id": "asyncimg_0123456789abcdef",
  "query_url": "https://api.example.com/v1/images/tasks_async/asyncimg_0123456789abcdef"
}
```

## 6. BB 任务查询

```http
GET /v1/images/tasks_async/{task_id}
Authorization: Bearer <提交任务的同一个 API_KEY>
```

### 排队中

```json
{
  "status": "queued",
  "task_id": "asyncimg_0123456789abcdef"
}
```

### 处理中

含调用上游、上传 OSS、等待计费确认等阶段：

```json
{
  "status": "processing",
  "task_id": "asyncimg_0123456789abcdef"
}
```

### 成功

```json
{
  "status": "succeeded",
  "task_id": "asyncimg_0123456789abcdef",
  "data": [
    {
      "url": "https://cdn.example.com/images/results/output-1.png"
    }
  ]
}
```

### 失败

```json
{
  "status": "failed",
  "fail_reason": "image generation failed"
}
```

成功与失败查询均返回 HTTP `200`。仅当 `status === "succeeded"` 时可消费 `data[].url`。

## 7. SC / Gemini

### 7.1 上传参考图（可选）

文生图不必上传。图生图且无安全可访问的 HTTPS URL 时，可先上传：

```bash
curl -X POST 'https://api.example.com/v1/uploads/images_sc' \
  -H 'Authorization: Bearer sk-...' \
  -H 'Idempotency-Key: upload-reference-001' \
  -F 'file=@reference.png'
```

成功 `200 OK`：

```json
{
  "url": "https://storage.example.com/images/inputs/1/asyncimg_0123456789abcdef.png",
  "filename": "reference.png",
  "content_type": "image/png",
  "bytes": 204800,
  "created_at": 1784548800
}
```

将返回的 `url` 填入后续提交的 `image_urls`。幂等重放成功时可能带响应头 `X-Idempotency-Replayed: true`。

常见上传错误：

| HTTP | `error.type`（示意） | 说明 |
|---|---|---|
| `429` | `async_image_upload_rate_limited` | 限频，通常带 `Retry-After: 60` |
| `409` | `async_image_upload_byte_quota` | 输入字节额度不足 |
| `409` | `async_image_upload_idempotency_conflict` | 同幂等键不同请求 |
| `409` | `async_image_upload_in_progress` | 相同上传仍在处理 |
| `409` | `async_image_upload_result_unavailable` | 旧结果已失效，需换新幂等键 |
| `413` | — | 单文件过大 |
| `503` | — | 存储 / 准入不可用 |

### 7.2 提交生图

```http
POST /v1/images/generations_sc
Authorization: Bearer <GEMINI_GROUP_API_KEY>
Content-Type: application/json
Idempotency-Key: sc-20260720-001
```

文生图：

```json
{
  "model": "gemini-3-pro-image-preview",
  "prompt": "现代客厅，北欧风，自然光",
  "resolution": "2K",
  "aspect_ratio": "16:9"
}
```

图生图：

```json
{
  "model": "gemini-3-pro-image-preview",
  "prompt": "保留构图，把场景改成夜景",
  "image_urls": [
    "https://storage.example.com/images/inputs/1/asyncimg_0123456789abcdef.png"
  ],
  "resolution": "4K",
  "aspect_ratio": "auto"
}
```

| 字段 | 必填 | 说明 |
|---|---|---|
| `model` | 是 | 模型名 |
| `prompt` | 是 | 提示词 |
| `image_urls` | 否 | 参考图 URL 列表 |
| `resolution` | 否 | `1K` / `2K` / `4K`（`0.5K` 规则同 BB） |
| `aspect_ratio` | 否 | 规则同 BB Gemini |

成功受理返回 HTTP `200`，且 body 内 `code` 也为 `200`：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "asyncimg_0123456789abcdef",
    "status": "pending",
    "type": "image",
    "progress": 0
  }
}
```

### 7.3 查询任务

```http
GET /v1/tasks_sc/{task_id}
Authorization: Bearer <提交任务的同一个 API_KEY>
```

| `data.status` | 含义 | 建议 |
|---|---|---|
| `pending` | 排队 | 继续轮询 |
| `processing` | 处理中 | 继续轮询 |
| `completed` | 成功 | 消费 `result.images` |
| `failed` | 失败 | 停止并读错误 |

排队：

```json
{
  "code": 200,
  "data": {
    "id": "asyncimg_0123456789abcdef",
    "status": "pending",
    "progress": 0,
    "type": "image"
  }
}
```

处理中：

```json
{
  "code": 200,
  "data": {
    "id": "asyncimg_0123456789abcdef",
    "status": "processing",
    "progress": 60,
    "type": "image"
  }
}
```

成功：

```json
{
  "code": 200,
  "data": {
    "id": "asyncimg_0123456789abcdef",
    "status": "completed",
    "progress": 100,
    "type": "image",
    "result": {
      "images": [
        {
          "url": [
            "https://storage.example.com/images/results/output-1.png"
          ],
          "expires_at": 1784552400
        }
      ],
      "videos": []
    }
  }
}
```

- `result.images[].url` 始终为**字符串数组**。
- 公开 CDN 直链时 `expires_at` 为 `0`；私有桶签名 URL 时为该次查询 URL 的 Unix 过期秒。
- `videos` 固定为空数组（结构兼容，本期不支持视频）。

失败：

```json
{
  "code": 200,
  "data": {
    "id": "asyncimg_0123456789abcdef",
    "status": "failed",
    "progress": 60,
    "type": "image",
    "error": {
      "message": "image generation failed",
      "type": "task_failed"
    },
    "failReason": "image generation failed"
  }
}
```

## 8. 对外状态对照

| 阶段 | BB `status` | SC `data.status` |
|---|---|---|
| 已入队 | `queued` | `pending` |
| 调用上游 / 上传 / 计费等中间态 | `processing` | `processing` |
| 成功（结果已落库且计费已确认） | `succeeded` | `completed` |
| 失败终态 | `failed` | `failed` |

注意：`processing` 可能表示图片已生成但仍在上传或结算，客户端须继续轮询，直到成功或失败终态。

## 9. 错误响应

### 9.1 常见 HTTP 状态

| HTTP | 场景 |
|---|---|
| `400` | 参数无效、`stream=true`、非法尺寸/比例、上传文件无效等 |
| `401` | API Key 无效 |
| `403` | 平台不匹配，或普通生图 / 异步生图未开启 |
| `404` | 任务不存在、方言不匹配，或非提交所用 API Key |
| `409` | 幂等冲突 / 上传冲突 / 输入字节额度不足 |
| `413` | 请求体、上传或参考图超限 |
| `429` | SC 上传限频（常带 `Retry-After: 60`） |
| `503` | 对象存储、运行配置或上传准入不可用 |

任务查询的 `failed` 是业务终态，仍使用 HTTP `200`。上表非 `200` 用于请求本身无法被鉴权、验证或受理的情况。

### 9.2 BB 错误格式

```json
{
  "error": {
    "type": "async_image_generation_disabled",
    "code": "async_image_generation_disabled",
    "message": "asynchronous image generation is not enabled for this group"
  }
}
```

### 9.3 SC 错误格式

```json
{
  "code": 403,
  "message": "asynchronous image generation is not enabled for this group",
  "data": null,
  "error": {
    "type": "async_image_generation_disabled",
    "message": "asynchronous image generation is not enabled for this group"
  }
}
```

### 9.4 计费与参考图简要说明

- 计费复用分组现有图片规则；提交成功不等于最终一定执行或已扣费。
- 参考图仅允许 HTTPS 或受限 `data:image/...;base64,...`；禁止内网地址（防 SSRF）；单图默认上限约 32 MiB；可解码格式为 JPEG / PNG / GIF / WebP。

## 10. 附录：旧版 Redis 异步接口

仅 OpenAI / Grok 分组。与持久化体系**独立**，任务前缀为 `imgtask_*`。

| 方法 | 路径 | 别名 |
|---|---|---|
| `POST` | `/v1/images/generations/async` | `/images/generations/async` |
| `POST` | `/v1/images/edits/async` | `/images/edits/async` |
| `GET` | `/v1/images/tasks/{task_id}` | `/images/tasks/{task_id}` |

请求体与对应同步 Images 接口相同（JSON 或 multipart）；不支持 `stream: true`。

提交成功 `202`：

```json
{
  "id": "imgtask_0123456789abcdef",
  "task_id": "imgtask_0123456789abcdef",
  "object": "image.generation.task",
  "status": "processing",
  "created_at": 1784092800,
  "expires_at": 1784179200,
  "poll_url": "/v1/images/tasks/imgtask_0123456789abcdef"
}
```

查询状态：`processing` → `completed` / `failed`。成功时 `result` 对齐同步 Images 响应（`b64_json` 已剥离，图片走对象存储 URL）。任务默认约 24 小时过期，执行超时约 30 分钟。未启用对象存储时提交/查询可能返回 `404`。

详细英文说明见 `docs/ASYNC_IMAGE_TASKS.md`。

## 11. 快速对接清单

1. 确认分组已开启「图片生成」+「异步生图」，平台与路径匹配。
2. 按方言选择提交路径（Gemini 用 `_gm` / `_sc`，OpenAI 用 `_oa`）。
3. 建议携带 `Idempotency-Key`，避免网络重试重复计费。
4. 按 `Retry-After: 3`（约每 3 秒）轮询，以 body 内 `status` 为准。
5. 仅在 BB `succeeded` 或 SC `completed` 时下载图片 URL；签名 URL 可能过期，需重新查询获取新链接。
