## 日常发版（就这几步）

先确保改动已在 `main` 并已推送，GitHub Actions 已启用。然后：

```bash
# 确认已推到 main
git checkout main
git pull origin main

# 打 annotated tag（Release 说明会进 GitHub Release）
git tag -a v0.1.162 -m "Fork 发行版：一键安装/升级与完整 Release 产物"

# 推送 tag（这一步会触发 Actions 自动打包）
git push origin v0.1.162
```

下次发版只改版本号，例如上游到了 `0.1.164` 就打 `v0.1.164`。

然后打开：

1. https://github.com/JasonWangJie/sub2api/actions → 等 **Release** 绿勾  
2. https://github.com/JasonWangJie/sub2api/releases → 确认有：
   - `sub2api_0.1.162_linux_amd64.tar.gz`
   - `sub2api_0.1.162_linux_arm64.tar.gz`
   - `checksums.txt`

> CI 会按 tag 自动写入 `VERSION` 并上传产物，**不必**先手改 VERSION 再发版。

## 产物出来后怎么装

```bash
# 安装
curl -sSL https://raw.githubusercontent.com/JasonWangJie/sub2api/main/deploy/install.sh | sudo bash

# 升级
curl -sSL https://raw.githubusercontent.com/JasonWangJie/sub2api/main/deploy/upgrade.sh | sudo bash
```

目标机：Linux + PostgreSQL + Redis + sudo。不要 `git clone` / `go build` / `pnpm install`。

常用：

```bash
sudo systemctl status sub2api
sudo journalctl -u sub2api -f
/opt/sub2api/sub2api --version
# 配置：/etc/sub2api/config.yaml
```

## 版本号说明（简要）

| 项 | 说明 |
|---|---|
| 源码文件 | `backend/cmd/server/VERSION`（如 `0.1.162`） |
| 发版开关 | Git tag `v0.1.162`（推送后自动打包） |
| 定版规则 | 合并原作者后**有版本就跟对方**；没有就自己定 |
| 注意 | tag 必须带 `v`；已存在的 tag 不要重复推 |

当前首次发行版：`v0.1.162`。

---

## 合并原作者后

1. 合并 `upstream`，按 `deploy/FORK_RELEASE.md` 保住 Fork 身份  
2. 有上游版本 → 打同号 tag；没有 → 自己定  
3. 再执行上面的「日常发版」四条命令  

---

## 常见问题

| 现象 | 处理 |
|---|---|
| Actions 要 Enable | 先启用再推 tag |
| 没有 `.tar.gz` | 不要勾选 simple_release |
| install 拉不到版本 | Release 还没跑完 |
